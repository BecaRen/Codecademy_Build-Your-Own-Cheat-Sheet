This cheat sheet is one of Codecademy's practice projects. It lists the HTML elements (with related attributes) used in the creation of forms that I've come across so far in my coding journey.

  1. Element Tags
  2. Attributes
Element Tags
Element Name Description Attributes
<form> Form The wrapper element for HTML forms.
Responisible for collecting information to send "somewhere else".		 action, method
<input> Input Creates an input field for the user to fill out. Self-closing. name, id, type, list, value, step
<label> Label Used to identify an <input> element.
It displays the text between its opening and closing tags. 		for
<select> Select Creates a dropdown list, an organised list from which the user can choose one item.
Requires that the choices be listed using the <option> element.
Useful in case there are many options to choose from.		 id, name
<datalist> Datalist Alternative to the dropdown list with a great number of options.
The <datalist> tag can help the user search through the list by typing characters in the input field.		 id
<option> Option Displays the options of a dropown list or a of datalist.
In the firts case, the text describing the option must be written between its opening and closing tags. In the second case, the text describing the option is given by the value attribute.		 value
<textarea> Textarea Allows to create a bigger text field (than <input type="text">). rows, cols
Red attributes are required, blue attributes can be omitted.



Attributes
Attribute Description
action Determines where the information is sent.
method The attribute is assigned a HTTP verb that is included in the HTTP request (i.e. POST or GET).
name Without this attribute the information in the input field won't be submitted.
At submission the values of name and value are paired together and sent where stated by the actionattribute
id Used wihtin the <input> tag so it can be uniquely linked to its <label> tag via the for attribute;
or within the <datalist> tag so it can be uniquely linked to its <input> tag via the list attribute.
type Describes what the kind of input is requested of the user.
Among its possible values: "text", "password", "number", "range", "checkbox", "radio", "submit".
list Used within the <input> tag when creating a datalist.
Links back to the id attribute in <datalist>.
value It's value becomes whatever the user has typed into the field.
If type=text or submit, the value can be omitted as it's implied.
If type=checkbox or radio, it's required.
Required also when using the <option> element.
step Used in case of type=number.
It allows to add arrows inside the input field to increase or decrease the value in the input field.
for Used within the <label> tag to uniquely link to an element carrying an id attribute.
rows Used determine the amount of rows we want in the textarea.
cols Used determine the amount of cols we want in the textarea.
